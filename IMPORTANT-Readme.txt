
⚠️ Les sauvegardes de ce GIT ne fontionnent pas en local !!!!!!!!!!!!

----------------------------------------------------------------------
VOUS NE POURREZ PAS VISUALISER CE SITE EN LOCAL VIA LE DEPOT GIT !!!
----------------------------------------------------------------------
Il sert uniquement de sauvegardes de versions mais est inutilisable 
en local. Les versions de ce dépôt Git sont des copies des dossiers 
et fichiers du serveur (transférés via commandes SSH). Ces dépôts 
fonctionnent seulement si push sur le serveur en ligne (car ils sont 
configurés avec les accès (BDD) du serveur, pas de BDD locale).
----------------------------------------------------------------------

Le projet a été directement développé sur le serveur d'hébergement
de PlanetHoster. 

Pour un gain de temps, nous avons utilisé les outils mis à notre 
disposition pour mettre en place un environnement WordPress fonctionnel
directement via le NOC (la plateforme d'administration de l'hébergeur) 
en quelques clics. 

C'est l'hébergeur qui à notre demande a généré l'environnement 
Wordpress directement sur le serveur (générant en auto la BDD, les
accès principaux... 

Comme tout le site sera effectué via la panneau admin Wordpress, c'est
un gain de temps de le modifier directement sur le serveur (ça évite
de perdre du temps à le faire en local et de devoir paramétrer pour le 
transférer et le mettre en ligne ensuite sur le serveur de l'hébergeur).

L'environnement, comme le site (thème, pages, contenus...) sera donc
uniquement géré en ligne. Nous nous sommes juste contentés de récupérer
les dossiers et fichiers en local pour sauvegarder différentes étapes
du projet.

( La version gratuite de l'hébergeur étant limitée, je me suis juste 
contenté d'effectuer des add, commit, push etc via SSH. Mais il est 
impossible de visualiser le site dans l'état en local. Vous pourrez
seulement voir le code via le Git. Pour voir le site (en ligne), 
allez voir le lien donné dans le dossier de présentation...).
